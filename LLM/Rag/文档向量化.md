# 文档向量化（Embedding）

## What — 是什么

把文本变成数字向量。语义相近的文本，向量距离近。

```
"Java 线程池怎么用"         → [0.12, -0.34, 0.87, ...]  (1024个浮点数)
"ThreadPoolExecutor 用法"   → [0.11, -0.31, 0.89, ...]  ← 距离近
"今天天气不错"              → [-0.78, 0.45, -0.12, ...]  ← 距离远
```

检索就是：query 向量化 → 在向量库里找距离最近的 K 个 → 返回对应文档块。

## Why — 为什么

### 为什么不能直接搜关键词

- "学 Java" 和 "Java 入门教程" —— 没有共同关键词，关键词匹配搜不到
- "线程池" 和 "ThreadPool" —— 同义词，关键词匹配不知道它们是一个东西
- "后天呢" —— 口语化，关键词匹配无法理解上下文

向量化解决的是**语义匹配**，不是字面匹配。

### 为什么用 BGE-M3

一次推理同时输出两种向量，解决不同问题：

| | 稠密向量 (Dense) | 稀疏向量 (Sparse) |
|---|-----------------|-------------------|
| 数据 | 1024 个浮点数 | 词级权重字典 |
| 擅长 | 语义相似 | 关键词精准匹配 |
| 例子 | "学Java" 能搜到 "Java入门教程" | "Kafka" 不会搜出 "RabbitMQ" |
| 短板 | 专有名词区分弱 | 同义词替换不行 |

查询时两个向量一起搜，效果互补。

### 为什么 query 和 document 必须用同一个模型

模型 A 的 `[0.12, -0.34]` 和模型 B 的 `[0.12, -0.34]` 坐标含义完全不同。换模型 = 换坐标系 = 之前存的所有向量都得重建。

### 为什么选 BGE-M3 而不是 OpenAI Embedding

| | BGE-M3 | OpenAI Embedding |
|---|--------|-----------------|
| 费用 | 免费，本地跑 | 按 token 收费 |
| 中文 | 专门优化 | 通用，中文一般 |
| 稀疏向量 | ✓ 同时输出 | ✗ 只有稠密 |

## How — 怎么做

### 模型加载

```python
from FlagEmbedding import BGEM3FlagModel

model = BGEM3FlagModel(
    "BAAI/bge-m3",
    use_fp16=True,        # 半精度，内存减半
    devices=["cpu"]        # Mac M2 无 CUDA，只能 CPU
)
```

### 编码

```python
output = model.encode(
    ["Java 线程池怎么用"],
    batch_size=8,          # 一次处理几条。CPU 设 8-16
    max_length=512,         # 最长 token 数
    return_dense=True,      # 要稠密向量
    return_sparse=True,     # 要稀疏向量
)

dense = output["dense_vecs"][0]        # [0.12, -0.34, ...] 1024维
sparse = output["lexical_weights"][0]  # {"Java": 0.87, "线程池": 0.92, ...}
```

### 存入 Milvus

```python
{
    "id": "doc_0_parent_1_child_2",
    "dense_vector": [0.12, -0.34, ..., 0.05],
    "sparse_vector": {"Java": 0.87, "线程": 0.92},
    "parent_id": "doc_0_parent_1",
    "parent_content": "## 核心参数\n\nThreadPoolExecutor...",
    "source": "java",
}
```

### 检索

```python
output = model.encode([query])

# 混合检索：稠密 + 稀疏一起搜
results = milvus.hybrid_search(
    dense_vector=output["dense_vecs"][0],
    sparse_vector=output["lexical_weights"][0],
    limit=5,
    ranker=WeightedRanker(1.0, 0.7)  # 稠密:稀疏权重 = 1:0.7
)
```

### 稠密向量的训练原理

Bi-Encoder + 对比学习：

```
训练时给模型看三个东西：
  query:     "线程池参数"
  正例 doc:  "corePoolSize 控制核心线程数"  → 拉近
  负例 doc:  "今天天气不错"                 → 推远
  负例 doc:  "红烧肉的做法"                 → 推远

模型通过大量这种三组训练，学会：语义相关的拉近，不相关的推远。
```

### 稀疏向量的原理

输出词级权重，类似 BM25 但权重是模型学的而不是统计的。`"Kafka"` 和 `"RabbitMQ"` 在稠密向量空间里很接近（都是消息队列），但稀疏向量能靠词级权重区分开。

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 检索不相关 | query 太短，稀疏 | HyDE：先生成假设答案再搜 |
| 专有名词搜不到 | 稠密向量对术语不敏感 | 加大稀疏权重 |
| CPU 编码慢 | batch_size 太大或太小 | 调至 16-32 |
| 向量不准 | 通用模型对领域词表示差 | 微调 BGE-M3（需要标注数据） |
