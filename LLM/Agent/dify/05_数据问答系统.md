# 基于Dify构建公司内部数据问答系统

## 概述

前面已经介绍了Dify接入知识库，也能接入外部的知识库ragflow，作为智能体开发平台，还能接入更多的工具，本课件介绍Dify接入MCP服务端。

### 项目背景

公司内部有大量的人事、行政和财务的问题，企业内部员工经常碰到各种问题需要咨询，导致内部人事、行政、财务等人员压力很大。本项目基于公司考勤数据、内部员工手册、等数据搭建一套智能体系统，重点解决下面问题：

- 应对各类考勤问题统计，实现自然语言的问答系统

- 员工手册实现员工各类问题咨询

除此之外本项目也能扩展到其他项目背景，例如面对企业内部订单数据的实时问答系统，售后问答等。

### mysql数据库

需要安装mysql数据库，如果已经安装好的直接修改配置就行。这里提供docker的安装方式，请确保能够正常使用docker环境。

在任意目录下创建空目录，名称为mysqld，把下面的命令复制到docker\-compose\.yaml文件中

```YAML
version: '3.8'

services:
  mysql:
    image: mysql:8.0.43
    container_name: mysql_container
    environment:
      MYSQL_ROOT_PASSWORD: 123456abc
      MYSQL_DATABASE: function_calls
      MYSQL_USER: dev
      MYSQL_PASSWORD: 123456
    ports:
      - "3306:3306"
    command: --local-infile=1
    volumes:
      - ./mysql_data:/var/lib/mysql

volumes:
  mysql_data:
```

注意上面的mysql的root的密码是“123456abc”，并且创建了用户“dev”密码为“123456”。

运行docker命令启动服务，第一次启动会下载官方镜像到本地，具体命令可以参考 [docker的常用命令介绍](https://vinctchanx.feishu.cn/wiki/FqxowZEZGiOgt2kaXY5cbMLAn0f)

```Shell
docker compose up -d 
```
