# 3\.3 LangChain企业员工手册问答系统



## V0\.2 版本

### 功能需求

- 需要前端页面，前端页面包括对话页面和知识库管理页面

- 支持用户注册和登录功能

- 支持用户历史对话

- 对话页面支持选择检索的知识库，模型的配置参数，配置参数包括模型名称，温度等参数

- 知识库页面支持知识库的预览，知识块的编辑和删除功能，修改以后的知识块需要重新解析，更新到向量数据库

- 支持代码版本管理



### 技术栈

- 数据库采用mysql

- 向量数据库使用milvus

- 大模型支持API调用，可以使用千问的在线API

- 前端使用vue，后端使用fastapi框架

- git版本管理

## V0\.1 版本

### 功能需求

- 实现基于公司内部员工手册的问答系统

- 实现从0\-1搭建，初步搭建离线知识库，然后实现用户的基于离线知识库的问答系统

- 对话页面系统

### 技术栈

- langchain

- qwen/ollama

- chromadb



### 提示词

需要你实现基于员工手册的问答系统，数据放在data目录下。需要搭建rag系统，要求如下：

1. 使用的技术栈包括。langchain, ollama, chromadb。用到大模型和embedding模型，你需要检查本地的ollama模型，使用ollama支持的模型就行，尽量不下载新模型

    1. 需要搭建离线知识库和在线问答系统，完成完整的RAG系统开发

3. 使用vue写一个前端对话页面，问答系统支持流式输出。交互页面也设计美观简洁，色调年轻化，尽量使用浅蓝色风格，加上以上渐变色。

4. 后端使用fastapi完成接口开发，代码要规范。可读性要好

5. 项目代码管理使用uv，创建好离线python环境
