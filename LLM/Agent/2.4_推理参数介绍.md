## 概述

### 1\.1 推理（Inference）

- 在参数固定的情况下，根据输入生成输出。其中的输入就是常说的提示词prompt

与之对应的就是训练或者微调，需要通过学习大量的数据，来调整模型的权重。**推理是不改变模型的权重参数的。前面讲了可以通过优化提示词来提升模型的回答效果，这里使用优化推理的参数，来提升模型的回答效果。**

### 1\.2 概率分布

常说的模型，本质上是一种概率分布。例如对于Bert分类模型来说，如果是3分类的话，输入内容$x$是文本序列，输出的分类为$y\in \{C_1, C_2, C_3\}$，输出$y$的取值就是对应的分类标签。

把模型输出的概率列表，称为概率分布，每个位置对应了分类标签。例如上面的例子，可见Bert模型是一种概率分布模型。同样的道理，大语言模型也是概率分布模型，输入prompt，每次推理输出下一个token的概率分布，根据一定的策略获取下一个token，如此循环往复直到处罚结束条件以后，大模型会停止输出。

用数学公式表示如下

$p(x_t | x_{<t}) \in \mathbb R ^{|V|}$

其中$|V|$表示词典的个数。获取下一个token的方法就是本节内容要介绍的推理参数。

## 推理过程

上面已经介绍了大模型的生成内容的过程，输入提示词以后，模型输出概率分布，然后从概率分布中获取一个token作为下一个token，拼接在提示词后面，一直循环输出直到**触发结束条件**，这个过程就是推理过程也称为解码过程 decode。接下来分别介绍推理的参数。

推理过程的整体流程图如下，可以看到每个参数调节的位置。

### 2\.1 Top P

对输出的概率分布按照概率大小降序排列，前面多个token的概率之和等于p，从这个token集合中随机取一个token作为输出，就是top p采样。详细见上面的图。

### 2\.2 温度参数

温度参数首先影响模型输出的logits，之后再进行softmax概率归一化，再结合其他的推理参数使用。

- 温度越低（小于1）：整体概率分布会变得陡峭，模型的输出变得“严谨”。主要原因过低的T，使得高概率的token概率值进一步变大，意味着高概率的token被采样出来，所有模型的输出变得“保守”。

- 温度越高（大于1）：整体概率分布变得平缓，模型的输出变得更加“天马行空”，甚至出现不知所云。主要原因是过大的T，使得每个token的输出概率基本差不多，意味着低概率的token被采样出来，过低概率的token被采样出来，会导致输出内容失去基本语义，变得不知所云。

### 2\.3 最大输出token数

这个参数相对简单，输出的最长token数。输出最长token会受到模型支持的最大上下文限制，例如模型最大上下文限制为32K或者128K。真实的输出长度应该满足:

$input\_tokens + output\_tokens \le model\_context\_limit$

max\_tokens这个参数主要限制output\_tokens这个参数

### 2\.4 词频惩罚

根据 token 在已生成文本中出现的次数，进行线性或累积惩罚的机制，出现越多 → 惩罚越重

Token i 出现的次数为$c_i$，词频的惩罚系数为$\beta$

$z_i ' = z_i - \beta \cdot c_i$

其中$z_i$为token i输出的logits值，$z_i '$为惩罚以后的logits值。

### 2\.5 出现惩罚

只要某个 token 在已生成文本中出现过一次，就对它施加一次固定惩罚，鼓励模型引入“新词/新概念”。

Token i 在已生成的文本中是否出现过，如果出现过记为1，否则为0，用$t$来表示，惩罚系数为$\gamma$

$z_i' = z_i - \gamma \cdot t$

其中$z_i$为token i输出的logits值，$z_i '$为惩罚以后的logits值。

### 2\.6 Beam search集束搜索\*

集束搜索表示在推理的过程中，需要**保留最好的n条路径**，也就是一次性生成n个候选答案，最终可以输出概率值最高的一条，或者随机从n中选择一条。

这里需要理解推理解码的本质，生成文本的过程其实是从下面的路径中解码出一条“概率较好”的路径。

能够计算出所有的路径的概率，然后获取概率最大的几条路径，这样效果是最佳的。但是整个过程的计算空间过大，例如词表按照5万计算（已经远小于当前主流大模型的词表了）。生成1000字的文本，理论上共有路径数

$50000^{1000}$

计算规模过大，为了简化计算，每次解码保留概率路径最大的n条，使用动态规划可以求解上面的问题。

代码：

```Python
*#!/usr/bin/env python*
*# -*- coding: utf-8 -*-*
*"""*
*@Author  : Vincent*
*@Time    : 2026/1/18 16:24*
*@File    : _04_inference_params.py*
*@Function :*
*"""*
import os

from openai import OpenAI

client = OpenAI(
    base_url=os.getenv("BASE_URL"),
    api_key=os.getenv("API_KEY"))
chunks = client.chat.completions.create(
    model="qwen-flash",
    messages=[{"role": "user", "content": "如何学习大模型？"}],
    stream=False,
    n=3,
    frequency_penalty=1.2,
    presence_penalty=0.9,
    temperature=1.9,
    top_p=0.5,
    max_tokens=100,
)
answers = []

print(chunks.choices[0].message.content)
print("-------------------" * 5)
print(chunks.choices[1].message.content)
```
